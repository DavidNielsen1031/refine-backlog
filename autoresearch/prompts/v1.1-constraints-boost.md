# Prompt Variant v1.1 — Constraints Boost

## Hypothesis
The current gap guidance for has_constraints says "Add technical constraints, scope limits, or assumptions" which is too vague.
The scoring regex checks for: tags >= 2, OR assumptions array non-empty, OR /constraints|limitations|assumptions|rules|scope|out of scope/i in text.

## Change
Replace the has_constraints line in the gap guidance with a more explicit instruction that:
1. Tells the LLM to add a "Constraints:" or "Assumptions:" section header (triggers regex)
2. Tells the LLM to populate the structured.constraints array (triggers assumptions check)
3. Gives concrete examples

## Old Line
```
- "has_constraints": Add technical constraints, scope limits, or assumptions
```

## New Line
```
- "has_constraints": CRITICAL — Add a "Constraints:" section with 2-3 specific technical constraints, scope boundaries, or assumptions. Examples: "Constraints: Must work with existing PostgreSQL schema. Out of scope: mobile responsive design. Assumes: user has admin role." Also populate the constraints array in structured output.
```

## Expected Impact
- has_constraints: 14% → 50%+ (targeted boost)
- Other dimensions: should be neutral (change is isolated to constraints guidance)
