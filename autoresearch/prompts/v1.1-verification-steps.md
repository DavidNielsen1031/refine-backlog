# Prompt Variant v1.1 — Verification Steps Boost

## Hypothesis
The rewrite prompt's gap guidance section had entries for all 5 scored dimensions EXCEPT
has_verification_steps. This is the only dimension at 24% — all guided dimensions score 55-100%.
Adding explicit guidance with concrete examples should close the gap.

## Change (deployed to production)
Added to rewrite-prompt.ts gap guidance:
```
- "has_verification_steps": Add a "Verification:" section with 2-3 concrete steps to prove it works
  (e.g., "Run npm test and confirm all pass", "curl /api/health returns 200",
  "Open the page and verify the button is visible", "Assert that the database record exists")
```

## Expected Impact
- has_verification_steps: 24% → 60%+ (targeted boost)
- Other dimensions: neutral or slight improvement (more complete specs)
- Agent-ready: 58% → 70%+ (verification was the gating dimension)
